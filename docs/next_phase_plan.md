# 下一阶段开发计划-2026.4

## 开发目标

### 目标 1: 引入 GraphRAG 提升复杂问题回答准确率

#### 问题背景
当前 RAG 系统在处理复杂问题时存在以下局限：
- 仅依赖语义相似度检索，难以捕捉实体间的深层关系
- 无法理解问题的推理链条和依赖关系
- 多跳问题（需要多步推理的问题）回答准确率较低

#### GraphRAG 解决方案
GraphRAG（Graph-based RAG）通过构建知识图谱增强检索能力：
- **实体提取**：从文档中识别并链接实体（人名、地点、概念等）
- **关系建模**：建立实体间的关联关系
- **社区检测**：通过图算法发现紧密相关的概念群组
- **全局检索**：利用知识图谱的社区结构进行高层级总结

#### 初步开发步骤

```
Step 1: 知识图谱构建
├── 选定图数据库（Neo4j / Milvus Graph / TuGraph）
├── 设计实体Schema（政策文件中的关键实体类型）
├── 编写实体抽取Prompt（利用LLM提取三元组）
└── 实现批量抽取流程

Step 2: GraphRAG 检索实现
├── 图数据库写入接口封装
├── 混合检索策略（向量+图谱）
├── 子图检索与剪枝策略
└── 检索结果与原始文档关联

Step 3: 与现有系统集成
├── 向量检索 → GraphRAG 分层检索
├── 检索结果融合排序
└── 性能基准测试
```

#### 技术选型
| 组件 | 推荐方案 | 说明 |
|------|---------|------|
| 图数据库 | Neo4j | 生态成熟， Cypher 查询方便 |
| 图嵌入 | graphrag-indexing | Microsoft 开源的 GraphRAG 工具包 |
| LLM 调用 | 复用现有 DashScope | 保持一致性 |

### 目标 2: 使用 LangGraph 重构项目架构

#### 重构背景
当前项目基于 LangChain 的简单 Agent 封装，随着功能增加：
- 工具调用链路不清晰
- 状态管理分散在各处
- 中间件机制较简陋，难以扩展

#### LangGraph 优势
- **有向图表达**：清晰定义 Agent 的工作流程
- **内置状态管理**：通过 StateGraph 统一管理对话状态
- **节点可复用**：各工具节点可独立测试和复用
- **支持条件分支**：复杂对话逻辑更容易表达

#### 重构思路

```
现有架构:
  User → ReactAgent.execute() → Tools → Response

重构后架构:
  User → StateGraph
           │
           ├── Node: user_input（接收用户输入）
           ├── Node: retrieve（知识库检索）
           ├── Node: reasoning（ReAct推理）
           ├── Node: tool_call（工具调用）
           ├── Node: response（生成回复）
           │
           └── Edge: 条件边（根据状态流转）
```

#### 初步开发步骤

```
Step 1: 状态定义
├── 定义 AgentState（消息历史、上下文、工具结果等）
├── 确定状态更新规则
└── 定义入口/出口状态

Step 2: 节点拆分
├── 将现有工具迁移为独立节点
├── 保持向后兼容（工具接口不变）
├── 每个节点独立测试
└── 设计节点错误处理机制

Step 3: 边与路由
├── 定义条件边（判断是否调用工具、是否结束等）
├── 设计回退机制（检索失败时的处理）
└── 绘制完整流程图

Step 4: 集成与迁移
├── 替换 ReactAgent 为 LangGraph Agent
├── 保持 API 接口不变（外部调用方无感知）
├── 性能对比测试
└── 逐步灰度切换
```

#### 关键设计考量
- **向后兼容**：现有 `app.py` 调用方式保持不变
- **增量迁移**：先在子模块试点，再全量迁移
- **日志与监控**：复用现有的 middleware 日志机制
- **状态持久化**：考虑对话历史的持久化方案

## 开发优先级建议

```
Phase 1（短期）: LangGraph 重构
├── 风险较低，改动可控
├── 可提升代码可维护性
└── 为后续 GraphRAG 打好架构基础

Phase 2（中期）: GraphRAG 引入
├── 需要数据准备和图数据库选型
├── 与重构后的 LangGraph 更好结合
└── 需要进行准确率对比实验
```
